# 이력서 기반 채용공고 매칭 가이드

`job-posting-match`는 구직자의 이력서/경력요약과 희망조건을 바탕으로 잡코리아·사람인 공개 채용공고를 검색하고, 공고별 적합도와 지원 전략을 정리하는 읽기 전용 스킬입니다.

## 사용 예시

```text
내 이력서 기준으로 지원할 만한 채용공고를 찾아줘.

이력서 요약:
- 퍼포먼스 마케팅 5년
- GA4, Google Ads, Meta Ads, SQL, Looker Studio 사용
- B2C 커머스 캠페인 운영
- CRM/리텐션 캠페인 경험

희망조건:
- 서울
- 퍼포먼스 마케터/그로스 마케터/CRM 마케터
- 보험영업, 대출영업, 텔레마케팅 제외
- 상위 10개만 추천
```

## 에이전트가 하는 일

1. 이력서에서 직무, 스킬, 산업, 경력 연차, 희망 지역, 제외 조건을 추출합니다.
2. 검색어를 1~3개로 나눠 잡코리아와 사람인 공개 검색 결과를 조회합니다.
3. 공고 제목, 회사, 목록 요약, 공개 링크를 기준으로 매칭 점수를 계산합니다.
4. 추천 이유, 주의점, 이력서/자기소개서에서 강조할 포인트를 정리합니다.

## 직접 실행 예시

```bash
python3 job-posting-match/scripts/job_posting_match.py   --resume-text "퍼포먼스 마케터 5년. GA4, Google Ads, Meta Ads, SQL, 커머스 경험. 서울 희망."   --location 서울   --negative 보험영업   --negative 대출영업   --limit 10
```

JSON으로 받고 싶으면:

```bash
python3 job-posting-match/scripts/job_posting_match.py   --resume-file ./resume.txt   --keyword "퍼포먼스 마케터 GA4"   --keyword "그로스 마케터 SQL"   --location 서울   --json
```

## 결과 해석

점수는 합격 가능성 보장이 아니라 공개 공고 요약과 이력서 키워드의 적합도 신호입니다.

- 80점 이상: 우선 검토할 만한 공고
- 60~79점: 조건 확인 후 검토할 공고
- 60점 미만: 키워드 일치가 적거나 제외 조건 가능성이 있는 공고

## 안전 경계

이 스킬은 읽기 전용입니다.

- 입사지원, 지원서 제출, 파일 업로드, 개인정보 입력을 자동으로 하지 않습니다.
- 로그인, 스크랩/관심공고 저장, 메시지 발송을 자동으로 하지 않습니다.
- 이력서의 연락처, 주소, 주민등록번호 등 민감정보를 저장하지 않습니다.
- 연봉, 고용형태, 마감일, 세부 자격요건은 원문 공고에서 최종 확인해야 합니다.

## 한계

- v1은 공개 검색 결과 중심이라 상세 공고 본문 전체를 항상 읽지는 않습니다.
- 사이트 HTML 구조가 바뀌면 파서 업데이트가 필요할 수 있습니다.
- 같은 공고가 여러 사이트에 중복 노출될 수 있습니다.
